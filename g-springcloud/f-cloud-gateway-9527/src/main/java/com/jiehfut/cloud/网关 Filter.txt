网关使用 Filter（过滤器，拦截器） 的场景
1.请求鉴权
2.异常处理
3.统计 controller 信息
4.......

全局默认过滤器
单一内置过滤器（网关过滤器，38个，主要用于单一路由或者路由分组）
自定义过滤器


1.网关的高级特性
    - 路由
        - GateWay Handler Mapping
    - 断言 - 判断某个请求是不是匹配该路由（路径 请求方法 请求头 参数 时间 自定义条件）
        - 常用断言 api
        - 自定义断言工厂
        - path before after between cookie header host method
    - 过滤 - 断言后对匹配成功的请求进行处理（前置过滤器 - 参数校验 日志输出 权限校验 协议转换 流量监控）（后置过滤器 - 响应内容 响应头修改 日志输出 流量监控）
        - 全局默认过滤器
        - 单一内置过滤器（38个）
        - 添加，移除，修改 请求头 请求参数 响应头 等
        - 自定义过滤器


在 Spring Cloud Gateway中，一个请求到达网关后，会遍历所有的路由（Route），
对每个路由，会检查该路由的断言（Predicate）是否匹配。
如果某个路由的断言匹配成功，那么该请求就会进入这个路由对应的过滤器链（Filter Chain）。
如果没有任何路由的断言匹配成功，那么网关会返回一个404 Not Found
